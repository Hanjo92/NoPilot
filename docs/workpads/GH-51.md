## Progress

- Added `nopilot.commitMessage.customPrompt` to the extension manifest with `{{diff}}` and `{{language}}` placeholder support.
- Updated commit message generation so a non-empty custom prompt overrides the built-in `conventional` and `simple` format instructions.
- Preserved NoPilot's output contract by still requiring the provider to return only the commit message text.
- Wired the new setting through settings-panel state and the settings webview with a multiline textarea control.
- Added dedicated webview styling for the multiline custom-prompt textarea so the new control renders cleanly and resizes vertically on desktop and mobile widths.
- Excluded the root-level `GH-*` tracker handoff files from `.vscodeignore` so the extension package will not ship issue-specific finish helpers or comment artifacts.
- Added an `Unreleased` changelog entry for the custom commit-message prompt feature so the user-facing release notes match the implemented functionality.
- Added a concrete README example for `nopilot.commitMessage.customPrompt` so users can copy a working template with `{{language}}` and `{{diff}}` placeholders directly from the docs.
- Added an inline settings note in the commit-message section of the webview so placeholder usage and format-override behavior are explained where the custom prompt is edited.
- Added inline code styling for placeholder tokens inside settings notes so `{{diff}}` and `{{language}}` stand out clearly in the commit-message guidance.
- Extended prompt, manifest, settings-state, and settings-webview tests to cover the new behavior.
- Added `commitMessageGenerator.source.test.ts` so the entrypoint that reads `commitMessage.customPrompt` is covered directly, not only through prompt-helper tests.
- Added `GH-51-comment.md`, `GH-51-finish.sh`, and `GH-51-finish.test.sh` so the issue comment and `Done` label can be synced as soon as GitHub access is restored.
